# Artists Hub

## Назва проєкту

**Artists Hub** — вебзастосунок для перегляду інформації про музичних
виконавців, їх жанри та відгуки користувачів.

## Про проєкт і яку задачу він вирішує

Проєкт допомагає швидко знаходити популярних артистів та переглядати:

- базову інформацію про виконавців;
- короткий опис/біографію;
- жанри;
- додаткові деталі у модальному вікні;
- відгуки користувачів.

Основна задача проєкту — надати зручний інтерфейс для знайомства з артистами в
одному місці, без необхідності вручну шукати дані на різних сайтах.

## Використані технології

- **HTML5** (компонентна структура через partials)
- **CSS3** (адаптивна верстка, окремі стилі секцій)
- **JavaScript (ES Modules)**
- **Vite** (розробка та збірка)
- Бібліотеки:
  - `axios` — HTTP-запити до API
  - `swiper` — слайдери
  - `izitoast` — повідомлення/нотифікації
  - `tui-pagination`, `raty-js` — допоміжні UI-компоненти

## Як запустити проєкт локально

### 1) Передумови

- Встановлений **Node.js** (рекомендовано LTS)
- **npm** (встановлюється разом із Node.js)

### 2) Встановлення залежностей

```bash
npm install
```

### 3) Запуск у режимі розробки

```bash
npm run dev
```

Після запуску відкрити у браузері адресу, яку покаже Vite (зазвичай
`http://localhost:5173`).

## Розгортання

Проєкт налаштований на збірку через Vite та може бути розгорнутий на **GitHub
Pages**.

Для коректної роботи на GitHub Pages у `package.json` вже вказано base-шлях:

```json
"build": "vite build --base=/artists-hub/"
```

## Структура проєкту (скорочено)

- `src/index.html` — основна сторінка
- `src/partials/` — HTML-частини сторінки (header, hero, artists тощо)
- `src/js/` — JS-логіка (API, рендер, модалки, слайдери)
- `src/css/` — стилі (спільні, утиліти, стилі секцій)

## Супутня інформація

- Проєкт використовує модульну структуру коду для зручного масштабування.
- Дані про артистів завантажуються динамічно через API.
- Інтерфейс адаптований під різні розміри екранів.
